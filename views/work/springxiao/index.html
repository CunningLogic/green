QQ钱包
  • IOS钱包SDK开发
  • 3月2014-至今
  • 团队20-30人
  • Objective-C, C/C++

S   ituation：QQ钱包是腾讯根据市场竞争需要，依托QQ亿级平台，利用QQ积累的用户社交关系链，发力移动支付、互联网金融、O2O等支付场景的战略性产品。

T   ask：[功能实现]：随着业务的变化与发展，根据产品同事设计的需求，在项目进度内完成开发任务； [性能调优]：手Q作为一个亿级平台，再小的问题也会被无限放大，因此对代码运行的性能要求也极其苛刻，优化代码、缩短响应时间、提升运行效率，来为用户提供更好的体验；[模块重构]：业务的变化与发展，不可避免的需要运用软件架构思想对相应的模块甚至是整个架构进行重构与调整。

A   ction：[安全策略]：无安全，不支付，支付SDK中运用了多种策略来保障资金安全，如对称加密算法AES、DES、3DES,非对称加密算法RSA,哈希算法MD5、SHA,数字证书等，客户端和服务器共同协作来保证支付安全；[软件架构]：项目中使用了传统的MVC架构，服务器返回的业务数据更新Modal,ViewController获取Modal数据赋值给View来展示，但存在一个问题，随着界面业务分支越来越多ViewController变得很庞大，这里还待优化；[数据存储]：IOS的沙盒机制在一定程度上保证了数据的安全，但对敏感数据进行加密存储是少不了的，虽然不能100%保证绝对安全，但可以提高被攻击的门槛；[技术预研]：终端机器更新换代飞快，支付方式也再不断的推陈出新，从原始的密码支付，到现在的指纹支付、声波支付、扫码支付、NFC支付、蓝牙支付、刷脸支付，寻找可行、可靠、便捷的支付方式，我们从未停止。

R   esult：从2014年1月份上线第一个版本到2015年8月份用户累计绑定银行卡已突破1亿张，除了支付、转账基础功能之外，产生社会影响力较大的产品还包括QQ红包、微粒贷、指纹支付、扫码支付等。京东、大众点评、美丽说等一批线上线下商户已接入手Q支付。

易物魔镜
  • Android客户端开发
  • 5-7月 2013
  • 团队3人
  • Java

S   ituation：大学时期的创业项目，应用定位“为大学生提供吃喝玩乐一条龙服务”。后台使用的是Windows系统+.Net 4+ SQL Server 2008数据库+IIS7服务器，由宏图志远|潘宏飞 完成。我主要做的是Android终端部分。

T   ask：终端部分，主要包括:[商品展示模块]：网站商品移动端展示；[摇一摇模块]：摇晃Android手机可以获取网站推荐的商品信息和图片；[定位模块]：用户双方输入用户名，就可以准确显示两者之间距离，并且不断刷新；[语音输入模块]：将用户的语音信息转换成相应的文字信息。

A   ction：[功能与组件]：按照我们最初的想法采用基站定位的确定用户的经纬度，后面发现该方式定位精度非常不理想，最后采用了调用第三方百度地图API解决此问题；[性能优化]：在UI线程中整合各项功能时需要开辟多个子线程，同时线程之间还需要数据交互，反反复复的在线程的AsynTask和Handler、Timer、TimerTask机制中摸索，才理清了各线程之间的关系；[模块重构]：刚开始对MVC架构理解不到位，将很多业务逻辑放在了Activity类中，导致Activity类越来越庞大，重用行、扩展性较低，维护代价也较大，后期将Activity类中业务相关的代码抽象到业务类中管理，降低了代码的耦合性。

R   esult：对市场缺乏了解，以及没有足够的运营能力，该项目遗憾的以失败结尾，但却收获了志同道合的伙伴。从技术角度来看基本实现了Android终端和网站最初的设计功能。
